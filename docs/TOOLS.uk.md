# MCP-інструменти

[Англійська версія](TOOLS.md)

OneAIWorkers дає AI-помічнику невеликий набір інструментів.

AI-помічник памʼятає, порівнює й вирішує. Worker тільки виконує дію.

## `hub_info`

Показує, що вміє OneAIWorkers і які додаткові сервіси налаштовані.

## Читання і перевірка

### `fetch_url`

Читає одну публічну HTTPS-сторінку і повертає текст.

Використовуйте для того, щоб:

- стежити за змінами на сторінці;
- перевіряти ціни конкурентів;
- читати публічну документацію;
- перевіряти публічну JSON або текстову адресу.

### `fetch_many_urls`

Читає до 10 публічних HTTPS-сторінок за один виклик.

Використовуйте для кількох сторінок або конкурентів.

### `fetch_rss`

Читає RSS або Atom-стрічку і повертає останні записи.

Використовуйте для новин, блогів, списків змін або досліджень.

### `check_url_status`

Перевіряє, чи працює публічне посилання і скільки часу займає відповідь.

Використовуйте для перевірки сайту або API.

## Дії

### `send_notification`

Надсилає повідомлення в Telegram, Discord, Slack або webhook.

### `call_webhook`

Викликає HTTPS webhook з JSON-даними.

Використовуйте для запуску Make, Zapier, n8n або вашої системи.

## Створення Worker

### `create_child_worker_from_template`

Створює дочірній Cloudflare Worker із безпечного шаблону.

Поточний шаблон:

- `webhook-forwarder`: пересилає POST, PUT і PATCH запити до налаштованої HTTPS-адреси.

Це не обовʼязково. Для цього потрібні облікові дані Cloudflare API. Він не виконує довільний код.

## Типовий порядок роботи

```text
Запуск AI за розкладом
  -> прочитати або перевірити дані через OneAIWorkers
  -> порівняти з памʼяттю AI
  -> вирішити, що важливо
  -> надіслати повідомлення або викликати webhook
  -> запамʼятати результат
```
