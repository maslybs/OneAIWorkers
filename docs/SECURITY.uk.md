# Нотатки з безпеки

[Англійська версія](SECURITY.md)

OneAIWorkers дає AI-помічнику змогу читати публічні сторінки й викликати зовнішні сервіси. Ставтесь до цього як до справжньої автоматизації.

## Безпечні початкові налаштування

- Він не виконує довільний код від AI.
- Дочірні Workers можуть використовувати тільки готові шаблони.
- Інструменти для публічних посилань блокують локальні й приватні адреси.
- Секрети треба зберігати як Cloudflare Worker secrets.
- Результати інструментів приховують чутливі поля в URL: `token`, `key`, `secret`, `password`, `auth`, `signature`.
- D1 використовується для OAuth, реєстру конекторів, дій конекторів і записів аудиту. Вона не використовується для памʼяті AI.

## Доступ до MCP

Для особистого використання задайте `MCP_SHARED_SECRET`.

Worker приймає його одним із цих способів:

- `Authorization: Bearer <secret>`;
- `x-oneaiworkers-token: <secret>`;
- `?key=<secret>` або `?access_token=<secret>`.

Варіант `?key=` простий, але менш безпечний, бо посилання можуть потрапляти в журнали або історію браузера.

Для публічних застосунків краще використовувати OAuth або Cloudflare Access.

## Cloudflare API token

Додавайте `CF_API_TOKEN` тільки якщо потрібно створювати дочірні Workers.

Використовуйте обмежений API token. Не використовуйте глобальний Cloudflare API key.

Token має зберігатися як секрет Worker.

## Попередження про дочірній Worker

Перший шаблон дочірнього Worker — `webhook-forwarder`.

Він зберігає цільове webhook-посилання в коді створеного Worker. Не використовуйте дуже чутливі webhook-посилання, поки не буде додано сильнішу роботу із секретами.

## Повідомлення і webhook

Інструменти для повідомлень можуть надсилати реальні повідомлення й викликати реальні сервіси.

Рекомендований порядок:

1. Прочитати або перевірити дані.
2. Дати AI-помічнику вирішити, чи це важливо.
3. Надсилати повідомлення або викликати webhook тільки коли правило чітке.
4. Дати AI-помічнику запамʼятати, що сталося.

## Не використовуйте в першій версії для

- оплат або повернень коштів;
- видалення робочих даних;
- автоматичного надсилання листів клієнтам;
- публікації в соцмережах без перевірки людиною;
- виконання довільного коду, створеного AI.
